## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag gap inside tag row, character counter gap |
| sm | 8px | Input label-to-field gap, tag internal horizontal padding, card meta row gaps |
| md | 16px | Default element spacing within a card, form field vertical gap, textarea internal padding |
| lg | 24px | Card internal padding, section padding within the capture page |
| xl | 32px | Gap between card grid rows, gap between textarea block and card grid |
| 2xl | 48px | Major section breaks (paste area vs. confirmation area) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Touch targets for accept/reject card buttons minimum 44px height (accessibility; WCAG 2.5.5).

**Source:** Existing pages use `p-6` (24px), `px-16` (64px), `gap-6` (24px), `py-32` (128px — too large for a functional page; reduced for capture UI). Scale inferred from Tailwind utility patterns in `page.tsx` and `LoginForm.tsx`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (normal) | 1.5 |
| Label | 14px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.3 |

**Notes:**

- Heading (20px / 600 / 1.3): used for page section titles ("粘贴内容" area label, "提取结果" section heading), card item title.
- Display (28px / 700 / 1.2): used for page-level heading only ("捕获知识").

**Source:** Existing code uses `text-2xl font-bold` (24px/700) for auth form heading, `text-sm font-medium` (14px/500) for labels, `text-lg leading-8` (18px) for body. Adjusted to tighter 4-size system. `text-3xl font-semibold` (30px/600) confirmed in `page.tsx` h1 — normalised to 28px display for capture page consistency.

## Component Specifications

### 1. Capture Page Layout (`/capture`)

```
max-w-3xl mx-auto px-6 py-16
  ├── Page heading (Display / 28px / bold)
  ├── Paste Input Block (full width)
  │   ├── Textarea (min-h: 200px, resize-y)
  │   ├── Character counter (Label / 14px / right-aligned, gray-400)
  │   └── Primary CTA button (full width, h-12, rounded-lg, bg-blue-600)
  └── Confirmation Cards Block (appears after extraction, replaces CTA with "确认并保存" button)
      ├── Section heading (Heading / 20px)
      ├── Card grid (1 column on mobile, 1 column on desktop — linear list per D-09 "clean, uncluttered")
      │   └── Card (each extracted item — see Card spec below)
      └── Bulk action row: "全部保留" text link + "确认并保存" primary button
```

### 2. TextPasteInput Component

- Textarea: `w-full min-h-[200px] resize-y px-4 py-4 text-base leading-relaxed border border-gray-300 dark:border-gray-600 rounded-lg bg-white dark:bg-gray-800 focus:ring-2 focus:ring-blue-500 focus:border-transparent outline-none transition-colors`
- Placeholder text: "粘贴你正在学习的文章内容……" (Body / 16px / gray-400)
- Character counter: bottom-right of textarea container, Label / 14px, gray-400. Format: "1,234 / 100,000 字符". Turns red-600 when over limit.
- Optional source URL field: single-line input below textarea, Label / 14px, placeholder "来源网址（可选）"
- Submit button: `w-full h-12 rounded-lg bg-blue-600 hover:bg-blue-700 text-white font-semibold text-base transition-colors`. Copy: "提取知识". Disabled + spinner state during extraction (replace label with "正在提取…").

### 3. Confirmation Card Component

Each extracted knowledge item renders as a card:

```
Card container: rounded-xl border border-gray-200 dark:border-gray-700 bg-gray-50 dark:bg-gray-900 p-6
  ├── Card header row
  │   ├── Title (Heading / 20px / semibold, gray-900 / white)
  │   └── Action row: "丢弃" text button (red-600) | "编辑" text button (gray-500)
  ├── Content (Body / 16px / 1.5 line-height, gray-700 / gray-300, max 3 lines expanded)
  ├── Meta row (Label / 14px / gray-500)
  │   ├── Domain badge: rounded-full bg-blue-100 text-blue-700 px-2 py-0.5 text-xs font-medium
  │   ├── Tags: same badge style, gray-100/gray-700 color
  │   └── Source (if present): truncated URL, gray-400, underline on hover
  └── Accept state indicator: unchecked by default, checkmark appears when "保留" is clicked
```

**Card states:**
- Default: shown above with no highlight
- Accepted: left border 4px solid blue-600, slightly elevated shadow (`shadow-sm`)
- Rejected: opacity-40, strikethrough on title, "已丢弃" label replaces action buttons (user can undo within session)
- Edit mode: title/content/source/tags fields become editable inline; see Edit Form spec below

### 4. Card Edit Form (inline, expands card)

Field order:
1. 标题 (Title): single-line text input, full width
2. 内容摘要 (Content): textarea, min 3 rows, 100-200 character guidance shown as helper text below
3. 领域 (Domain): single text input, pre-populated by AI
4. 标签 (Tags): tag input — type and press Enter or comma to add a tag; each tag renders as a removable pill (x button on right); existing tags shown inline

Tag pill style: `inline-flex items-center gap-1 rounded-full bg-gray-100 dark:bg-gray-700 px-3 py-1 text-sm font-medium text-gray-700 dark:text-gray-300`

Tag remove button: `×` character, 16px, gray-400, hover red-600.

Edit form action buttons: "取消" (text, gray-500) + "保存修改" (filled, blue-600, h-10 rounded-lg), right-aligned.

### 5. Loading and Processing States

**During extraction (after submit):**
- Textarea and submit button disabled
- Submit button shows inline spinner (16px animated border-top spinner, white on blue-600) + "正在提取…" text
- Below textarea: progress hint text (Label / 14px / gray-400): "AI 正在分析内容，通常需要 5-15 秒……"

**Skeleton cards (while streaming results if multiple chunks):**
- Show 2-3 skeleton card placeholders: rounded-xl border bg-gray-50 with animated pulse bars for title/content/tags

## Interaction Contracts

### Capture Flow State Machine

```
idle
  → (user pastes + clicks "提取知识") → extracting
extracting
  → (API success, items > 0) → confirming
  → (API success, items = 0) → idle (show "no items" error inline)
  → (API error) → idle (show error inline, textarea re-enabled)
confirming
  → (user accepts/rejects/edits cards)
  → (user clicks "确认并保存") → saving
saving
  → (success) → idle (show success toast, clear textarea and cards)
  → (error) → confirming (show error, allow retry)
```

### Tag Input Interaction

- Type to add tag. Commit on: `Enter` key, `,` (comma), `Tab` key.
- Delete last tag on `Backspace` when input is empty.
- Maximum 10 tags per item (soft limit; show warning at 10, prevent at 11).
- No autocomplete in Phase 1.

### Card Accept/Reject

- Accepting a card: single click "保留". No confirmation required.
- Rejecting a card: click "丢弃" → show inline confirmation below action buttons ("确认丢弃？ 是 / 取消"). On confirm: card collapses to rejected state with undo link (10-second window). No modal.
- "全部保留" bulk action: marks all non-rejected cards as accepted instantly.
- The "确认并保存" button is disabled if zero cards are accepted (all rejected or none confirmed). Shows tooltip: "请至少保留一条知识点".

### Character Counter

- Visible at all times once user starts typing.
- Format: "1,234 / 100,000 字符"
- Color transitions: gray-400 (0-90%), amber-600 (90-99%), red-600 (100%+).
- Submit button disables and turns gray-400 when over limit.
